Практическая работа № 22
"Создание библиотеки классов"

Цель работы:
Закрепление полученных знаний и навыков по работе классами в языке программирования С#

Общее задание:

  1. Используя интегрированную среду разработки (IDE) Microsoft Visual Studio создать 2 проекта: библиотеку классов (DLL) и WinForm/Console приложение для доступа к этой библиотеке
  2. Название пространства имен (Namespace) в библиотке классов должно соответствовать фамилии студента выполнившего ПР. Имя класса в пространстве имен должно соответствовать номеру ПР. Названия методов в классе должны соответствовать именованию типа TaskNN(), где NN - номер выполняемого задания в Вашем варианте. В 3-ем задании всех варантов название функции можно оставить в качестве имени метода.
  3. Входные параметры (один или несколько) необходимо вводить в формы/консоли (приложение) и передавать дальше в метод. Все реализуемые методы должны распологаться в DLL - библиотеки классов. Каждый метод должен принимать один или несколько параметров, в зависимости от задания. Каждый метод должен возвращять итоговое одно значение, еслм в задании не прописано множественное возвращение. Названия главных пространств имен (Namespace) в библитеке классов и в приложении WinForm/Console обязательно должны быть разные.
  4. В задании, где результатом работы должен быть логический ответ, "да" или "нет", использовать тип данных - Bool.
  5. ВНИМАНИЕ!!! Во всех заданиях варианта начальные значения должны вводиться с формы, используя соответствующие контроллы. Выходные значения (ответы) во всех заданиях варианта необходимо выводить в другие контролы на форму
  6. Интерфейс программы должен быть четко продуман, control'ы Visual Studio (TextBox/Label/RadioButton/CheckBox/MenuStrip/...) при построении интерфейса должны распологаться "не хаотично" и не как попало, а представлять полностью законченный дизайн формы и приложения (ПР) в целом. Для прототипирования интерфейса рекомендуется использовать средства MS Visio или бумажный элемент прототипа.
  7. Сдать проект, показав полностью готовую библиотеку классов и интерфейс программы с выполненными заданиями исходя из Вашего варианта
  8. Код на языке C# должен быть отформатирован по стилю оформления редактора кода в MS Visual Studio (никаких "лесенок" из кода быть не должно) и содержать комментарии в исходном коде при решении алгоритма.
  9. Предусмотреть контроль ввода начальных значений на корректность/правильность (например, выход за диапозон указанных чисел/значений, деление на 0, ввод только положительных/отрицательных значений, ...).
  10. Номер варианта для выполнения практической работы соответствует номеру студента/обучающегося по списку в учебном журнале группы.
  11. Материалы проекта обязательно сохранить на сервере в своей папке и в личном облачном хранилище.
