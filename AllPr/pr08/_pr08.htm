Практическая работа № 08
"Методы (функции) в языке С#"

Цель работы:
Получить первичные навыки по работе с функциями в языке программирования С#

Общее задание:

  1. Используя интегрированную среду разработки (IDE) Microsoft Visual Studio или JetBrains Rider создать проект и написать программы оформленные в виде отдельных функций (методов) для каждого задания из варианта, использующие консольный ввод/вывод и переменные целого типа и/или типа с плавающей запятой.
  2. Каждое задание из варианта должно быть реализовано в виде функции. Вызываться эти функции должны из главной функции Main() по указанию абонента, используя конструкцию ветвления switch-case. И главная функция и Ваши реализованные функций должны находиться в одном файле. Каждая функция должна принимать один или несколько параметров, в зависимости от задания. Каждая функция должна возвращать результат работы через оператор языка С# - return. Входные данные в задания должны вводиться через консоль.
  3. В задачках, где требуется работа с массивами обязательно вводить и сам размер массива, и значения элементов массива с консоли, предоставив пользователю возможность случайного заполнения, если не написано другое условие. Массив должен состоять не менее чем из 30 элементов. Заполнение массива должно происходить в функции Main(), и после передаваться в функцию в виде аргумента.
  4. Предусмотреть контроль ввода начальных значений на корректность/правильность (например, выход за диапозон указанных чисел/значений, деление на 0, ввод только положительных/отрицательных значений, количество цифр в вводимом значении и т.д.).
  5. Сдать проект, показав построенную цепочку команд в отладчике, убрав при этом все "точки остановки" отладчика, кроме последней. Во время защиты (сдачи) практической работы преподаватель может изменить начальный набор входных данных, в целях проверки корректности написанного кода.
  6. Код на языке C# должен быть отформатирован по стилю оформления редактора кода в MS Visual Studio или JetBrains Rider (никаких "лесенок" из кода быть не должно) и содержать комментарии в исходном коде при решении алгоритма.
  7. Номер варианта для выполнения практической работы соответствует номеру студента/обучающегося по списку в учебном журнале.
  8. Материалы проекта обязательно сохранить на сервере в своей папке и в личном облачном хранилище.
