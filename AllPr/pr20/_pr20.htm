Практическая работа № 20
"Unit-тесты"

Цель работы:
Получить первичные навыки по работе с Unit-тестами в языке программирования С#

Задание:

  • Используя интегрированную среду разработки (IDE) Microsoft Visual Studio требуется разработать класс для хранения и обработки данных с использованием unit-теста в соответствии с номером Вашего варианта.
  • Имя класса(ов) в проекте обязательно должно соответствовать номеру задания из Вашего варианта (taskХХ, где ХХ - номер Вашего задания из варианта)! Все классы должны быть в разных файлах проекта! Для выбора имени namespace использовать номер Вашего варианта (например, namespace var35)
  • Практическую работу выполнять в полном соответсвии по разобранным/рассмотренным на лекциях примерах!
  • Для комментирования исходного кода проекта необходимо обязательно использовать документирующие комментарии языка C# (param, returns, summary)
  • Для каждого задания необходимо написать по 4 различных unit-теста, каждый unit-тест должен покрывать возможные потенциальные ошибки во время выполнения программы. Одинаковые тесты писать запрещено
  • Обязательно предусмотреть контроль ввода начальных значений на корректность/правильность (например, выход за диапозон указанных чисел/значений, деление на 0, ввод только положительных/отрицательных значений, ...) используя механизм обработки исключительных ситуаций!!!!!
  • Номер варианта для выполнения практической работы соответствует номеру студента/обучающегося по списку в учебном журнале.
  • Материалы проекта сохранить на сервере в своей папке и в личном облачном хранилище.

Продолжительность работы:
2 академических часа

Критерии оценивания выполненной работы:

  • три (3) любых правильно выполненных задания своего варианта на оценку "3" (удовлетворительно)
  • четыре (4) любых правильно выполненных задания своего варианта на оценку "4" (хорошо)
  • пять (5) правильно выполненных заданий своего варианта на оценку "5" (отлично)

ВАРИАНТ № А1/Б26
