Практическая работа № 14
"Работа со структурами и перечислениями в C#"

Цель работы:
Получить первичные навыки по работе с структурами и перечислениями в языке программирования С#

Общее задание:

  1. Используя интегрированную среду разработки (IDE) Microsoft Visual Studio создать проект и написать линейные программы (задание-ввод данных в консоли-ответ), использующие консольный ввод-вывод и переменные целого типа и/или типа с плавающей запятой.
  2. Разработать формат текстового файла, содержащего данные согласно варианта (не менее 20 уникальных строк). Основной функционал приложения должен обеспечивать: а) просмотр на консоли всех/нескольких записей по указанному номеру (прямое чтение файла); б) просмотр на консоли записей по заданному значению параметра поиска из варианта (последовательное чтение файла). Параметры поиска вводятся с консоли. Вызываться эти функции должны из главной функции Main() по указанию абонента!
  3. Все функции должны работать со структурами данных или перечислениями. Вторая функция может найти и выдать более одной записи или не найти вообще ничего. На консоль должны быть выданы все поля найденных записей.
  4. Предусмотреть контроль ввода начальных значений на корректность/правильность (например, выход за диапозон указанных чисел/значений, деление на 0, ввод только положительных/отрицательных значений, количество цифр в вводимом значении и т.д.). Вывод сообщений об ошибках выводить на консоль.
  5. Код на языке C# должен быть отформатирован по стилю оформления редактора кода в MS Visual Studio (никаких "лесенок" из кода быть не должно) и содержать комментарии в исходном коде при решении алгоритма.
  6. Предусмотреть контроль ввода начальных значений на корректность/правильность (например, выход за диапозон указанных чисел/значений, деление на 0, ввод только положительных/отрицательных значений, ...).
  7. Номер варианта для выполнения практической работы соответствует номеру студента/обучающегося по списку в учебном журнале группы.
  8. Материалы проекта обязательно сохранить на сервере в своей папке и в личном облачном хранилище.
